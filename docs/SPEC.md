## 仕様

### 画面レイアウト

- タイトルバー: 一番上の細長い領域。Linux の GUIで共通の、中央にタイトルが表示され、右側に最小化や閉じるボタンなどがある
- メニューバー: タイトルバーの下の細長い領域。メニューが表示される。メニュー項目について別途記述する
- タブ: メニューバーの下の細長い領域。リポジトリごとに一つのタブを表示する。メイン領域で開いているリポジトリをハイライトする。
- ツールバー: タブの下の細長い領域。今のところツールボタンを載せる予定はないが標準構成として作っておく。左端に、リポジトリとブランチを画像と同様の形式で表示
- メイン領域: ツールバーの下の、ウィンドウの大部分を占める領域。
  - 左ペイン: メイン領域の左側のやや狭い領域。local, remote, worktree の内容を、それぞれツリー表示。ペイン全体の表示非表示を切り替えられる。
  - 中央メイン: メイン領域の大部分を占める。デフォルトでグラフを表示する。
  - diff オーバーレイ: 右ペイン内のファイルリストでファイルが選択されたとき、left + center を覆う形で diff を表示する。右ペイン自体は残す。Esc キーで diff を閉じ、DAG 表示に戻り、ファイル選択も解除する。
  - 右ペイン: メイン領域の右側の領域。右ペインは、DAG 上でコミットが選択されている時と未コミットが選択されている時で異なる。
  - (コミット選択時): コミット専用レイアウトを表示する
    - 上部: コミットメッセージ領域。1 行目は大きめのフォント、2 行目以降は小さめのフォントで表示する。長いメッセージはこの領域内でスクロールする。
    - 中間: 最小限のメタ情報領域。author / date / hash を表示する。
    - 下部: コミットされたファイルリスト。各行は最小限の `A/M/D/R` バッジ + path で表示し、この領域内でスクロールする。
  - (未コミット選択時): WIP 専用レイアウトを表示する
    - 上部: unstaged ファイルのリスト
    - 下部: staged ファイルのリスト
    - 各行は最小限の `A/M/D/R` バッジ + path で表示し、それぞれの領域内でスクロールする。
  - 右ペインの上下比率は、コミット選択時と未コミット選択時で別々に持つ。将来的には境界をドラッグして比率変更できるようにする。
- スタータスバー: メイン領域の下の細長い領域。今のところ表示するものはないが標準構成として作っておく。
- 複数リポジトリはタブで切り替え (MVPスコープ外)

### 機能スコープ（v1）

| 機能 | 内容 |
|---|---|
| DAG グラフ | カラフルなブランチ線、ダークモード、美しいレイアウト |
| コミット情報 | hash / author / date / message（グラフ行内） |
| 変更ファイル一覧 | 変更種別（A/M/D/R）付き |
| diff 表示 | unified diff、シンタックスハイライト、旧行番号＋新行番号の2列 |
| リポジトリ管理 | タブで複数開く |
| gitignore 対応 | `core.excludesFile`（グローバル gitignore）対応 |

フィルタ・検索は v2 以降。

### リポジトリを開く操作

| 方法 | 用途 |
|---|---|
| CLI 引数 `gitviewer /path/to/repo` | メイン。ターミナルから `gitviewer .` など |
| 起動時に前回セッションを復元 | 引数なし起動時のデフォルト動作 |
| 最近開いたリポジトリ（Recent） | セッション復元と統合して管理 |
| GUI から「フォルダを開く」ダイアログ | タブ追加ボタン [+] 等から起動 |
