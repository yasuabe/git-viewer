# ROADMAP

## フェーズ0: 部品単体開発

技術的リスクが高い2部品を、アプリ本体とは独立して先行開発・検証する。

| 部品 | 完成基準 |
|---|---|
| DAGビュー | ダミーデータでグラフ描画。コミットクリックで選択イベント発生（ハンドラはコンソール出力のみ） |
| diffビュー | ダミーデータでdiff描画。シンタックスハイライトあり |

---

## フェーズ1: MVP

フェーズ0の2部品を組み込み、実際のリポジトリで動作する最小構成。感触を確かめることが目的。

### スコープ

- タブ1つ
- DAGビュー + diffビューが実データで動作
- リポジトリをメニューから開く操作
- 左ペイン・右ペイン・ツールバー・ステータスバーの基本UI
- 設定ファイルなし

### スコープ外

- タブ複数対応
- 設定ファイル

---

## フェーズ2: GitKraken 閲覧機能相当

MVP の感触が良ければ、GitKraken の閲覧機能のみのバージョンと言えるレベルまで育てる。

### 追加スコープ

- リポジトリをメニューから開く（フォルダダイアログ）※前倒し実装済み
- タブ複数対応
- 左ペイン（LOCAL / REMOTE / WORKTREES ツリー表示）※基本 UI と表示制御は前倒し実装済み
- 設定ファイル（前回開いたリポジトリの復元など）
- `core.excludesFile` 対応

---

## 期限

既存クライアントを参照できる期間内に、主要な閲覧体験を固める。
