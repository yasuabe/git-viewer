# ADR-005: 右ペインの表示切替ロジック

- **ステータス**: 決定済み
- **決定日**: 2026-03-23
- **決定者**: yasuabe2613

---

## コンテキスト

右ペインの表示内容は `selectedCommit` の値によって切り替わる。
`selectedCommit` の型設計と初期値、および表示パターンを決定する。

---

## 決定1: `selectedCommit` の型

WIP ノードと通常コミットを区別するため、専用の判別型を使用する。
`null`（未選択）は持たない。リポジトリを開いた時点で常にいずれかが選択された状態とする。

```typescript
type SelectedCommit
  = { type: "commit"; hash: CommitHash }
  | { type: "wip" }
```

### 理由

- 型安全に WIP と通常コミットを区別できる
- AI によるコード生成時にミスが出にくい
- `null` を排除することで、右ペインの「空」状態を考慮しなくて良くなり、実装がシンプルになる

---

## 決定2: 初期値

リポジトリを開いた直後、または新しいタブに切り替えた直後の初期選択は以下の通り。

| 状態 | 初期値 |
|---|---|
| WIP ノードあり（未コミットの変更がある） | `{ type: "wip" }` |
| WIP ノードなし（クリーンな状態） | `{ type: "commit", hash: <最新コミットのhash> }` |

---

## 決定3: 表示パターン

```
selectedCommit
├── { type: "wip" }
│   ├── WIP 用右ペインを表示
│   ├── 上部: unstaged ファイルリスト
│   └── 下部: staged ファイルリスト
│
└── { type: "commit", hash }
    ├── commit 用右ペインを表示
    ├── 上部: コミットメッセージ
    ├── 中間: 最小限のメタ情報帯（author / date / hash）
    └── 下部: 変更ファイルリスト（A / M / D / R + path）
```

### 詳細ルール

- commit 選択時のメッセージ領域では、コミットメッセージ 1 行目を大きめのフォント、2 行目以降を小さめのフォントで表示する
- commit 選択時と WIP 選択時では、右ペインは同一レイアウトの中身差し替えではなく、別種の pane として扱う
- commit メッセージ領域、commit ファイルリスト領域、unstaged/staged ファイルリスト領域は、それぞれ独立にスクロールする
- 右ペインの上下比率は commit 用と WIP 用で別々に持ち、将来的にはドラッグで変更できるようにする
- ファイルリストでファイルが選択された場合、diff は中央ペイン内部ではなく left + center を覆うオーバーレイとして表示する
- diff 表示中に Esc キーを押すと、diff を閉じ、DAG 表示へ戻し、ファイル選択も解除する

---

## トレードオフ

| 観点 | 内容 |
|---|---|
| 未選択状態の排除 | 右ペインが常に何かを表示するため、空状態のUIを実装しなくて良い。一方、リポジトリが空（コミットが一件もない）の場合の扱いは別途検討が必要 |
| 判別型の採用 | 将来的に選択対象が増えた場合（タグ選択など）も型を拡張するだけで対応できる |
