# ADR-001: DAGレイアウトアルゴリズムの自前実装

- **ステータス**: 決定済み
- **決定日**: 2026-03-22
- **決定者**: yasuabe2613

---

## コンテキスト

Git Viewer の核心機能はコミットグラフ（DAG）の表示である。
GitKraken に近い「直線ブランチ」スタイル（同一ブランチのコミットが同一列に並ぶ）を採用する方針とした。

既存ライブラリを調査した結果、以下のことが判明した。

- **d3-dag**: 汎用 Sugiyama 法の実装。Gitコミットグラフ専用ではなく、メンテナンスモードに入っている。直線ブランチのレーン割り当てには非対応。
- **dagre**: 汎用グラフレイアウト。Git用レーンアルゴリズムを持たない。
- **Mermaid gitgraph**: 宣言的・静的な描画用。実用規模のリポジトリには非対応。

直線ブランチを実現している GUI クライアントは GitKraken と SourceTree のみだが、どちらも非 OSS であり参照実装が存在しない。

## 決定

DAGレイアウトアルゴリズム（レーン割り当て・エッジ描画）は自前実装とする。

## 理由

- 要件（直線ブランチ・ダークモード・美しい表示）を満たす既存ライブラリが存在しない
- Git コミットグラフ特有の制約（1行1コミット・時系列順・レーン再利用）により、汎用 DAG ライブラリは過剰かつ不適合
- アルゴリズムの核心は「レーン割り当て」と「エッジ引き回し」の2問題に絞られており、自前実装のスコープは限定的

## 実装方針（詳細は別途）

- コミット順序: Temporal Topological Sort（committer date 降順でのDFS）
- レーン割り当て: ブランチの生存期間を追跡し、同一ブランチを同一列に固定。マージ後に空き列を再利用
- エッジ種別: branch edge（縦線）と merge edge（斜め線）を区別して描画
- 描画レイヤー: 技術スタック確定後に決定（Canvas / SVG / WebGL）

## 影響・トレードオフ

| 観点 | 内容 |
|---|---|
| 実装コスト | レーン割り当てのヒューリスティック設計が必要。複雑なマージ履歴でのエッジ交差最小化は難しい |
| 品質リスク | GitKraken 同等の美しさに達するかは未確定 |
| 保守性 | 独自コードであるため長期的に自分でメンテする必要がある |
| 柔軟性 | 要件変化（曲線ブランチへの切替など）に対応しやすい |
