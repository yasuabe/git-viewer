# ADR-002: 技術スタック選定

- **ステータス**: 決定済み
- **決定日**: 2026-03-23
- **決定者**: yasuabe2613

---

## コンテキスト

開発者はバックエンド（純粋関数型言語）専門であり、スタンドアロンアプリの開発知見が少ない。
ソースコードの読み書きは基本的に AI に委ねる方針のため、以下の条件を優先して技術スタックを選定する。

- AI の学習データが豊富で、コード生成品質が高い
- スタンドアロンアプリ開発の実績が多く、ローリスク
- Linux 対応

## 決定

| 層 | 選定 |
|---|---|
| シェル | Electron |
| UI | React + TypeScript |
| グラフ描画 | Canvas 2D |
| git バックエンド | simple-git（Node.js ライブラリ） |
| diff ハイライト | diff2html または monaco-editor |

## 理由

**Electron + React + TypeScript**

Electron / React / TypeScript の組み合わせは、AI の学習データが最も豊富。AI によるコード生成品質が高く安定している。Tauri（Rust）は軽量で優秀だが Rust の学習データが相対的に少なく、AI 生成コードの品質にばらつきが出るリスクがある。Python + Qt はモダン UI 実装の知見が AI に少ない。

**Canvas 2D**

DAG グラフの描画には Canvas 2D を採用する。SVG はコミット数増加に伴い DOM 数が爆発しパフォーマンスが劣化する。WebGL は機能過剰。Canvas 2D は AI のサンプルコードも豊富で、ADR-001 で決定した自前レイアウトアルゴリズムとの統合が容易。

**simple-git**

システムにインストール済みの `git` コマンドをラップする Node.js ライブラリ。libgit2 系と比べて導入・利用がシンプルで、AI の習熟度も高い。`core.excludesFile` 等の git 設定も git コマンド経由で自然に尊重される。

## 影響・トレードオフ

| 観点 | 内容 |
|---|---|
| バンドルサイズ | Electron は Chromium を同梱するため配布バイナリが大きい（100MB 超）。許容する |
| メモリ使用量 | Electron は比較的重い。許容する |
| AI 親和性 | 本スタックは AI コード生成との相性が最も良く、開発リスクを低減できる |
| Linux 対応 | Electron は Linux を正式サポート |
